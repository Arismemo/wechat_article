# 微信原创合规说明

## 1. 微信平台的洗稿仲裁机制

微信公众平台对洗稿（抄袭性改写）设有专项投诉与仲裁流程。核心规则如下：

- **24 小时内收集到 ≥10 票仲裁票，且 ≥70% 的仲裁者认定构成洗稿**，平台即认定侵权成立。
- 认定成立后，**原创文章自动替换涉嫌洗稿的文章**（即侵权内容被原文覆盖），无需额外申诉。
- 即便文章**未标注原创标识**，仍可被认定为洗稿。

## 2. 平台明确标记的洗稿手法

以下改写方式被微信平台明确列为洗稿手法，产出内容一旦命中，极易触发投诉：

- 相同话题 + 相同切入角度 + 相同观点
- 近义词替换（用同义词逐句替换原文用词）
- 语序调整（重新排列句子成分而不改变意思）
- 段落顺序调整（重排原文段落顺序）
- 删除非关键词（删减修饰词后保留核心结构）

**仅进行以上表层改写，而不改变信息架构和论证角度，不构成合规改写。**

## 3. 本系统的相似度硬门禁

本系统在 Phase 4 流水线中内置了相似度硬门禁，通过 `phase4_similarity_max` 参数（默认值：0.45）控制。

门禁逻辑（`Phase4PipelineService._passes_thresholds`）：

- 当审稿报告中的 `similarity_score` 超过 `phase4_similarity_max` 时，即便 LLM 给出 `final_decision="pass"`，流水线也**不会自动推送草稿到微信**。
- 超出阈值的任务会被路由到 **人工审核（NEEDS_MANUAL_REVIEW）** 状态，等待运营人员介入。
- 仅当 `similarity_score ≤ phase4_similarity_max` 且其他指标（overall_score、policy_risk、factual_risk、ai_trace_score）均通过时，才允许自动推草稿。

## 4. 运营人员发布前须确认的事项

在将系统产出的草稿发布到微信之前，运营人员必须逐条确认以下两点：

1. **差异度确认**：稿件与原文在信息架构、论证角度上是否存在实质性差异，而非仅为表层改写。建议逐段对照原文，重点检查是否只做了近义词替换或段落顺序调整。

2. **来源授权确认**：稿件参考的原文素材是否已取得必要授权，或其内容是否属于可合理引用的公开信息（如官方文档、公开研究报告）。无授权转载他人受著作权保护的内容，即便改写后仍存在法律风险。

**系统的自动化门禁只能过滤相似度指标，无法替代人工的内容判断和版权审查。最终发布决策由运营人员负责。**
