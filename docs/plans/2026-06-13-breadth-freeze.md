# 决策:冻结上游广度扩张,先闭合质量回环 · 2026-06-13

> 状态:**生效中**。来源:本仓库自主评审(`docs/plans/2026-06-13-audit-remediation.md`)。

## 决策

**暂停**以下两个子系统的**功能扩张**,直到"质量度量回环"用真实数据证明核心写作链路有效:

- 选题情报层(`topic_*` 模型 / `topic_intelligence_service` / `/admin/topics`)
- 写作因子库(`factor` / `task_factor_usage` / `admin_factors*`)

"暂停扩张"= 不新增模型、不新增页面、不扩 API 面;**已有代码保留并维护**(bug 修复、随重构同步不受限)。

## 为什么

评审发现项目存在结构性精力错配:

1. **质量信号是断的**:`feedback_sync_provider` 默认 `disabled`(`app/settings.py`),真实读者反馈(打开/读完/涨粉)未回流。**系统当前无法判断自己产出的稿件好不好。**
2. **却在持续往上游加机器**:近期提交集中在选题情报(7 个新模型)与因子库,而非闭合上面的信号缺口。
3. 行业经验(2025–2026):内容/AI 产品在核心产出质量被真实数据验证前扩功能广度,是把投入压在未验证假设上。

在"产出是否有效"不可观测时,新增的选题/因子功能无法被证伪 —— 投入无法转化为可信结论。

## 解冻条件(Gate)

同时满足即可解冻:

- 质量回环上线(见整改计划 T5):真实或人工反馈指标按任务回流,后台可见单篇真实表现。
- 至少 1 个回流周期的数据表明:核心写作链路(Phase 3→4)产出的稿件,真实表现与审稿分存在可解释的正相关(或暴露出明确的质量短板供优先修复)。

满足后,基于真实数据重排选题/因子的优先级,而非凭直觉继续铺功能。

## 不受本冻结影响

- 可靠性整改(重试/DLQ/幂等/事务,T3a/T3b/T4)
- 合规硬约束(T6)
- 前端去字符串化(T7)
- 上述子系统的缺陷修复与随重构的同步改动
