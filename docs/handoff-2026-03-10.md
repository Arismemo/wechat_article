### 2.2 设置页关键回归修复

问题 1：设置页一直停在“等待加载”

- 根因：`/admin/settings` 页面内嵌 JS 在 Python 三引号模板里把 `"\n"` 展开成脚本真实换行，导致浏览器解析 `<script>` 报语法错，`loadAll()` 根本没执行。
- 修复：把 `.split("\n")` / `.join("\n")` 改为输出字面量 `\\n`（也就是 `.split("\\n")` / `.join("\\n")`），并新增页面源码断言防回归。
- 相关提交：`3013203`

问题 2：点击“测试连通性”后结果不直观可见 / 被刷新覆盖

- 根因：测试请求返回后被后续 `loadAll()` 的“页面快照输出”覆盖；且输出面板默认折叠。
- 修复：测试/告警/保存类动作完成后自动展开“调试输出”，并让 `loadAll()` 支持不覆盖动作结果。
- 相关提交：`50dfdce`
